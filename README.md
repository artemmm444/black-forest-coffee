# ЧЁРНЫЙ ЛЕС — товарный лендинг (спешелти-кофе)

Одностраничный лендинг под продажу обжаренного кофе. Редакторская, «студийная»
подача: моноширинная техническая обвязка (живые часы, номер партии), крупная
антиква, оксблад-красный акцент, полноэкранная кинематографичная съёмка,
острые углы. Спек-лист товара вместо карточек, таймер акции, форма заказа.

**Задача:** показать товар и довести до заявки.
**Стек:** чистые HTML + CSS + JavaScript (модули), без сборки и зависимостей.
Шрифты — Space Mono / Fraunces / Newsreader (Google Fonts), фото — Unsplash.

## Секции
Полноэкранный первый экран с манифестом · бегущая строка · спек-лист товара
с ценой · блок «почему мы» · галерея · отзывы · таймер акции + форма заказа ·
футер с контактами.

## Запуск локально
Нужен любой статический сервер (из-за ES-модулей `file://` не подойдёт):
```bash
python3 -m http.server 4551
# открыть http://localhost:4551
```

## Правка контента
Весь контент — в `config.js`: бренд, оффер и манифест, товар (цена, характеристики,
фото), выгоды, галерея, отзывы, акция (`endsInHours`, `stockLeft`), контакты.
Правишь один файл — получаешь новый лендинг под другой товар.

## Деплой на GitHub Pages
```bash
git init && git add . && git commit -m "coffee landing"
gh repo create black-forest-coffee --public --source=. --push
```
Затем **Settings → Pages → Branch: main /(root)** → живой адрес вида
`https://<username>.github.io/black-forest-coffee/`.

## Форма заказа
Сейчас это демо: валидация + подтверждение на экране (без бэкенда — GitHub Pages
статический). Чтобы заявки приходили вживую:
- **Formspree:** заменить обработчик `submit` в `main.js` на `fetch` к своему
  Formspree-эндпоинту (работает на статике, письма на почту).
- **Telegram:** развернуть на Netlify и использовать серверную функцию — заявки
  будут падать в Telegram.
