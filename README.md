# ЧЁРНЫЙ ЛЕС — товарный лендинг (спешелти-кофе)

Одностраничный лендинг под продажу одного товара — обжаренного кофе. Редакторская
подача: тёмно-зелёная палитра, крупная антиква (Fraunces), витрина товара,
таймер акции и форма заказа с валидацией.

**Задача:** показать товар и довести до заявки.
**Стек:** чистые HTML + CSS + JavaScript (модули), без сборки и зависимостей.
Шрифты — Google Fonts, фото — Unsplash.

## Секции
Первый экран с оффером · витрина товара с ценой и вкусовыми нотами ·
блок «почему мы» · галерея · отзывы · таймер акции + форма заказа · футер.

## Запуск локально
Нужен любой статический сервер (из-за ES-модулей `file://` не подойдёт):
```bash
python3 -m http.server 4551
# открыть http://localhost:4551
```

## Правка контента
Весь контент — в `config.js`: бренд, оффер, товар (цена, ноты, фото), выгоды,
галерея, отзывы, акция (`endsInHours`, `stockLeft`), контакты. Правишь один файл —
получаешь новый лендинг под другой товар.

## Деплой на GitHub Pages
```bash
git init && git add . && git commit -m "coffee landing"
gh repo create black-forest-coffee --public --source=. --push
```
Затем в репозитории: **Settings → Pages → Branch: main /(root)** → живой адрес
вида `https://<username>.github.io/black-forest-coffee/`.

## Форма заказа
Сейчас это демо: валидация + подтверждение на экране (без бэкенда — GitHub Pages
статический). Чтобы заявки приходили вживую:
- **Formspree:** заменить обработчик `submit` в `main.js` на `fetch` к своему
  Formspree-эндпоинту (работает на статике, письма на почту).
- **Telegram:** развернуть на Netlify и использовать функцию `lead.js`
  (как в проекте барбершопа) — заявки будут падать в Telegram.
